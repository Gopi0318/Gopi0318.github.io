XGB: Attention-CNN-LSTM + XGBoost Hybrid Forecasting

Project: two-stage forecasting pipeline (CNN+self-attention→LSTM embeddings → XGBoost). Reviewer: Gopala Krishna Kuchi

What this project is and what I implemented

This project builds a two-stage price forecasting pipeline: a 1D CNN + self-attention feeding an LSTM (trained on raw OHLCV windows) whose final hidden-state embeddings are refined by an XGBoost model to predict next-day returns/probabilities. My contributions (implemented and tested): data ingestion & windowing; CNN-Attention-LSTM model and training loop with checkpointing; embedding extraction and export; XGBoost hyperparameter tuning and model persistence; an inference wrapper that emits the specified JSON; Crew AI DecisionAgent and ExplainAgent; integration tests and a backtesting harness that simulates decisions and reports performance metrics. The repository, commits, and sprint records are linked below.

Project overview — quick facts

Acceptance targetswindowed OHLCV → embeddings → XGBoost; inference JSON: {date, cnn_lstm_score, xgb_prob}
Decision thresholdsxgb_prob > 0.6 → BUY, < 0.4 → SELL, otherwise HOLD
PerformanceEnd-to-end inference target ≤ 5 s on sample data
RobustnessMissing bars handled via imputation logic; model checkpoints and early stopping included
Agents
  • InferenceAgent — loads CNN+LSTM + XGB and returns scores.
  • DecisionAgent — maps xgb_prob → BUY/SELL/HOLD via Crew AI LLM prompt (outputs JSON only).
  • ExplainAgent — returns a concise rationale using feature importances.

What I completed (concise)
